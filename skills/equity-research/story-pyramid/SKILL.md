# 金字塔故事线

## 触发条件

用户有零散材料需要组织成有说服力的叙事，例如：
- "帮我把这些数据整理成汇报逻辑"
- "明天的客户汇报，帮我搭个主线"
- "这些材料很散，帮我梳理成一条线"

## 框架

采用 **SCQA** 框架（麦肯锡/芭芭拉·明托金字塔原理）：

### S (Situation) — 背景
- 读者已知的、不争的事实
- 设定场景，让听众点头说"对，就是这样"

### C (Complication) — 冲突
- 背景之下出现的矛盾、问题、挑战
- "但是……"——打破常态
- 这是钩子，决定听众是否愿意继续听

### Q (Question) — 核心问题
- 冲突引出的、必须回答的问题
- 通常就是一句话

### A (Answer) — 答案
- 你的核心主张/建议
- 下面用金字塔逻辑支撑

## 金字塔支撑层

Answer 之下的支撑结构：

**第一层：核心主张（1句话）**

**第二层：三大论据（3个）**
每个论据之间是 MECE 关系

**第三层：论据的证据链**
- 每个论据配 2-3 个数据点或案例
- 数据来源标注

## 输出格式

```
📐 金字塔故事线

S (背景)
[1-2句话，公认事实，任何人不会反对]

C (冲突)  
[1-2句话，发生了什么变化/挑战]

Q (核心问题)
[一句话，自然的疑问]

A (答案)
[你的核心主张]

━━━━━━━━━━━━━

论据一：[标题]
  ├─ 数据点1（来源：XXX）
  ├─ 数据点2
  └─ 案例：XXX

论据二：[标题]
  ├─ 数据点1
  └─ 数据点2

论据三：[标题]
  ├─ 数据点1
  └─ 案例：XXX

━━━━━━━━━━━━━

📑 PPT 页面建议
1. 封面
2. Executive Summary（S→C→Q→A 一页讲完）
3. 论据一（2-3页）
4. 论据二（2-3页）
5. 论据三（2-3页）
6. 实施路径 / 下一步
7. 附录
```

## 硬约束

- SCQA 每一步不超过 2-3 句话，不啰嗦
- C 必须是真的冲突，不是"为了更好地发展"这种废话
- 论据之间必须 MECE，重叠了要合并
- 如果材料和故事线对不上，宁可砍故事线也不编数据
- 每个数据点标注来源，来源不确定就说"待验证"
