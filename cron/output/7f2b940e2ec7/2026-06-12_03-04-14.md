## 📋 预安装代码审计协议

在安装任何新 Skill、MCP、依赖模块或第三方脚本之前，**必须先做静态审计**：

### 审计流程
1. **获取代码**：不要盲目执行 `curl | bash`。先使用 `curl`/`web_extract` 拉取 SKILL.md 和相关文件到本地
2. **完整静态扫描**：对文件内容进行正则/模式匹配检查
3. **重点监控二次下载行为**（供应链投毒的高发区）：
   - 包管理器：`npm install`、`pip install`、`brew install`、`cargo`、`gem`
   - 直接下载执行：`curl`、`wget`、`fetch()`、`urllib.request`
   - 绕过机制：`python -m http.server`、`ruby -e`、`git clone`
   - 混淆编码：`base64 -d | sh`、`eval()` 结合动态获取
4. **高风险文件格式警告**：
   - 编译二进制：ELF、.so、.dylib
   - 压缩包：`.tar.gz`、`.zip`、`.whl`
   - 可疑隐藏文件：以 `.` 开头的文件、包含大量不规则十六进制
5. **升级上报**：发现二次下载行为或高风险文件格式时，**必须强制中断安装**并向用户发出红色警报

**未通过安全审计的组件不得使用，无论其功能看起来多吸引人。**

## 🛡️ 权限收窄（系统级硬控制）

### 1. 核心文件权限保护
```bash
# 保护 Hermes 核心配置文件
chmod 600 ~/.hermes/.env
chmod 600 ~/.hermes/config.yaml
chmod 600 ~/.hermes/gateway.yaml

# macOS 不可变标志保护（类似 Linux chattr +i）
chflags uchg ~/.hermes/.env
chflags uchg ~/.hermes/config.yaml
```

### 2. 配置文件哈希基线
```bash
# 首次部署或确认安全后生成基线
sha256sum ~/.hermes/.env > ~/.hermes/.config-baseline.sha256
sha256sum ~/.hermes/config.yaml >> ~/.hermes/.config-baseline.sha256

# 审计时检查
sha256sum -c ~/.hermes/.config-baseline.sha256
```

### 3. 审计脚本保护
```bash
# 审计脚本自身加不可变锁
chflags uchg ~/.hermes/scripts/nightly-security-audit.sh
```

## ⚠️ 已知局限（请务必了解）

1. **LLM 认知层脆弱性**：Agent 的 LLM 认知层容易被精心构造的复杂文档绕过。**人工常识和二次确认是供应链投毒的最后防线**
2. **同 UID 读取**：Hermes 以当前用户身份运行，恶意代码同样具备该用户权限。`chmod 600` 无法防止同用户读取
3. **哈希基线非实时**：仅在夜间审计时检查，最大发现延迟约 24h
4. **审计推送依赖外部 API**：消息平台偶尔故障会导致推送失败，报告始终本地保存
5. **依赖 LLM 指令优先级解析**：`--light-context`（Hermes 隔离会话）是最有效的缓解方案，但根本上仍依赖 LLM 的指令优先级判断
6. **版本兼容性**：本指南基于当前 Hermes Agent 版本编写，未来版本可能引入原生安全机制导致部分措施过时或冲突
